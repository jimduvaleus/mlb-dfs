## Key Design Decisions

| Decision | Rationale |
|---|---|
| Exponential + Normal mixture | Exponential captures the heavy right tail (big-game upside); Normal captures typical day-to-day variance |
| PCA projection via 2Ã—2 linear system | Directly satisfies the `(Î¼', Ïƒ')` constraints rather than projecting a heuristic query point; exact solution within the PCA plane |
| Mixture CDF evaluated at empirical score grid | Discretises outputs to observed historical score values; the grid encodes which DK point totals are actually achievable |
| No sklearn dependency | PCA implemented with `np.linalg.svd` to stay within the existing NumPy/SciPy stack |
| Backward-compatible engine | `batter_pca_model=None` default preserves Phase 1-3 behaviour |

## Known Limitations / Phase 5 Inputs

- **PCA requires real historical data**: `scripts/fit_batter_pca.py` exits early if `historical_logs.parquet` is absent. The dummy copula data generated by `build_copula.py` does not populate batter-level per-game scores, so the fitting script requires actual Retrosheet-derived data.
- **No pitcher mixture model**: Pitchers continue to use a Gaussian marginal. A pitcher-specific model (e.g., a left-censored distribution capped at IP limits) is deferred to Phase 6 refinement.
- **Score grid is global**: All batters share one score grid derived from population-level historical scores. A slot-stratified grid (e.g., separate grids for leadoff vs. cleanup hitters) could improve tail accuracy and is a candidate Phase 6 enhancement.
